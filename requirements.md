# DIRETRIZES DE DESIGN

## Paleta de cores
A lógica aqui é equilibrar a identidade Dominus (laranja + preto) com um acento que ressoa com a história da Inglaterra (dourado). Fundo profundo escuro cria autoridade e premium sem precisar de imagem cara.
- Fundo principal: #0D0D0D (preto profundo)
- Fundo alternado de seção: #141414
- Laranja Dominus (CTAs e destaques): #FF6600
- Dourado suave (acento premium, bordas): #C9A84C
- Texto principal: #FFFFFF
- Texto secundário: #8A8A8A
- Bordas decorativas: 1–2px, laranja ou dourado

## Tipografia
Inter ou Montserrat, ambas gratuitas e geométricas. Headlines em ExtraBold/Black, corpo em Regular ou Medium. Nunca fonte serifada ou cursiva. O peso visual pesado no headline cria impacto imediato — a força está no contraste entre o título grande e bold e o corpo mais leve.

## Layout
Página única de rolagem vertical (scroll), full width. Seções bem espaçadas com padding generoso — não encher de conteúdo. Bordas finas em laranja ou dourado delimitando seções de destaque (o mesmo recurso visual do PDF da Cheidith funciona bem aqui). Hero com foto de imóvel de alto padrão ou Londrina ao fundo, com overlay escuro para garantir legibilidade do texto.

## Elementos visuais
Ícones minimalistas e lineares, sem arredondamento excessivo. Nenhum gradiente colorido — só gradiente escuro para profundidade. CTA sempre em laranja sólido com texto branco. A linha fina laranja como elemento decorativo de separação é o toque visual de identidade da Dominus.

## Tom visual geral
A página precisa parecer feita para a Inglaterra, não um template que poderia ser de qualquer cliente. O equilíbrio laranja + dourado cria exatamente essa leitura: "somos parceiros à altura da sua história."

## Assets
- Logos estão disponíveis na pasta `/public` (`logo-d.png`, `logo-institucional.png`, `logo-institucional-white.png`).
- Pode utilizar a logo white para o header.
- Use Lucide-react ou Radix UI Icons para ícones minimalistas.
- Pode usar `vite-init` para criar o projeto React/Tailwind no diretório atual se ele estiver vazio (exceto pela pasta public/ e este requirements.md). Mova o requirements e a pasta public para o lugar certo após a inicialização.
